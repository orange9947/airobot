# 零步轮输出断电实施计划

## 目标

修复 STM32 运动服务在单轮运动时给零步轮输出非零相位的问题，不改变协议、运动参数或非零轮相序。

## Task 1：锁定回归测试

修改 `tests/unit/test_robot_core.c` 的输出捕获器，使其记录左右两侧出现非零图案的次数。增加左右对称的单轮动作测试：活动轮必须至少输出一次非零相位，零步轮的非零计数必须始终为零，结束后左右最终输出必须为零。

先运行 `test_robot_core`，确认测试在当前实现上因零步轮被通电而失败。

## Task 2：最小修复

修改 `firmware/stm32/App/Motion/motion_service.c`。在调用 `motion->apply()` 前，根据左右目标绝对步数分别门控输出；目标为零时传入 `0`，目标非零时保留现有半步图案。

重新运行 `test_robot_core`，确认新增左右对称测试与既有双轮、超时、释放测试通过。

## Task 3：完整验证

运行完整 Python 测试、STM32 主机 C 构建与 CTest、STM32 交叉编译、ISA 检查和固件尺寸检查。运行 `git diff --check`，并确认受保护的第三方接入地址未出现在跟踪内容中。

## Task 4：提交与烧录

只暂存计划、运动服务和对应测试，自动提交并推送 `main`。烧录前确认电机 5V 仍断开并通知操作员 STM32、OLED 和 PC13 将复位；使用现有 ST-Link 流程烧录，随后恢复 ESP32 正式服务，保持 ESTOP。

## Task 5：真机复测

操作员重新确认 5.2V、轮子悬空、驱动灯全灭和可立即断电后，执行一次固定左轮 `+64` 半步动作。右侧 ULN2003 必须全程不亮；若左轮仍不转，停止动作并检查带载电压、ULN2003 输出和插头接触。
