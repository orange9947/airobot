# 左电机带载电压诊断实施计划

## Task 1：现场门禁

操作员当次确认万用表表笔已固定在左侧 ULN2003 `VCC-GND`、左右轮悬空、两板指示灯全灭、5V 已接通且可立即断开。未收到当次“已准备”不执行探针。

## Task 2：准备一次性探针

临时创建 `.codex-tmp/loaded_voltage_probe.py`，复用已部署的 `Stm32SpiLink` 和 `DeviceService`。探针固定使用左 10 步、右 0 步、10 half-steps/s、10 half-steps/s² 和 2000 ms 超时，不接受外部参数。主机语法检查通过后才允许运行。

## Task 3：执行唯一动作

探针恢复 USB 切换导致的链路丢失 ESTOP，确认 IDLE、零故障和零活动命令后进入 MANUAL，发送唯一一笔动作。执行期间操作员记录左侧 ULN2003 `VCC-GND` 的最低电压；右板必须全程不亮。命令拒绝、超时或异常时尝试 STOP，不重试。

## Task 4：恢复与清理

动作结束后请求 IDLE，确认左右完成步数为 10 和 0。重启 ESP32 恢复网页，保留随后因链路切换产生的 ESTOP，删除临时探针。根据操作员报告决定检查供电压降还是 ULN2003 输出与插头，不继续增加动作参数。
