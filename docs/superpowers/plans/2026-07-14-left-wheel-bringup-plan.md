# 左轮首次带电验证实施计划

- 日期：2026-07-14
- 设计依据：`docs/superpowers/specs/2026-07-14-left-wheel-bringup-design.md`
- 范围：静态预检、一次性 USB 探针、左轮单次低速动作、服务恢复与结果记录

## Task 1：锁定执行基线

步骤：

1. 确认工作树干净且 `main` 与 `origin/main` 同步，不夹带未知改动。
2. 确认 ESP32 仍使用 `/dev/ttyACM0`，端口未被其他串口程序占用。
3. 仅读检查当前的左轮 `PA8-PA11 -> IN1-IN4` 映射、半步序列、运动硬上限和线圈释放路径。
4. 运行现有 Python 和 C 主机测试，确认未运行真机动作时协议、状态机和运动完成路径仍通过。

验证命令：

```bash
.venv/bin/python -m unittest discover -s tests
cmake --build build/host
ctest --test-dir build/host --output-on-failure
git diff --check
```

## Task 2：准备固定的一次性探针

文件：

- 临时创建、执行后删除 `.codex-tmp/left_wheel_probe.py`
- 不修改 `firmware/esp32`、`firmware/stm32`、网页路由或 AI 工具

探针步骤：

1. 复用设备上已部署的 `Stm32SpiLink`、`DeviceService` 和生成的协议常量。
2. 启动 `DeviceService.run()`，等待链路建立和状态快照；输出只包含连接、状态、故障码、活动命令及 RX 错误的脱敏摘要。
3. 遇到 FAULT、链路不健康或 `active_command_id != 0` 时拒绝进入动作阶段。
4. 若只因 USB 切换进入 ESTOP，在当次人工确认后允许一次 `clear_estop()`；不解除 FAULT。
5. 如果无活动动作但当前是 MANUAL 或 AI，先请求 IDLE。随后必须确认 IDLE、故障码为零、活动命令为零。
6. 请求 MANUAL，仅调用一次 `move(64, 0, 120, 300, 1500)`；不接受命令行动作参数，不自动重试。
7. 只接受 `ok=true、left_steps=64、right_steps=0` 作为完成结果，然后请求 IDLE 并读取最终快照。
8. 任何异常都在链路可用时尝试一次 `stop()`，保留 ESTOP，不在异常处理中再调用 `clear_estop()`。
9. 探针必须通过主机语法编译检查，再允许上传执行；不将临时探针加入 Git。

## Task 3：执行前人工门禁

在调用 `mpremote run` 之前暂停，要求操作员当次再次回复“已准备”。该确认同时表示：

- 左右轮仍然悬空，旁边没有线材或手指接触轮子。
- ULN2003 5 V 断电手段仍可立即操作。
- 执行前 ULN2003 相位指示灯全熄灭，操作员正在观察左轮和两块 ULN2003 指示灯。

没有当次“已准备”，不运行探针。

## Task 4：执行单次动作

步骤：

1. 通过 `/dev/ttyACM0` 运行一次临时探针，不发送第二笔运动。
2. 完整捕获探针的初始快照、模式切换、动作结果、最终快照和异常信息；不输出或读取 Wi-Fi 口令、管理密码、API Key 或 `/config` 内容。
3. 如果操作员看到持续抖动、异响、不转或复位，立即断开 ULN2003 5 V；不等待探针自行收尾。
4. 无论协议成功还是失败，本阶段结束后都停止，等待操作员报告 `向前`、`向后`、`抖动` 或 `不动`。

## Task 5：恢复正式服务

步骤：

1. 软复位 ESP32，让现有 `main.py` 重新启动网络、SPI 轮询和 Web 服务。
2. 通过已知局域网 IP 做无凭据的 HTTP 可达性检查，不再打开 USB REPL 打断心跳。
3. 不通过复位 STM32 或调试脚本自动消除切换期间产生的 ESTOP。如网页显示 ESTOP，由操作员确认环境安全后使用已有的网页二次确认恢复。
4. 不在恢复流程中调用任何运动命令。

## Task 6：记录并提交结果

文件：

- 新增 `docs/testing/m1-left-wheel-bringup.md`

记录内容：

1. 固件和 Git 提交基线。
2. 固定的动作参数和探针返回值。
3. 初始与最终 STM32 状态、故障码、活动命令和 RX 错误数。
4. 操作员报告的方向/故障类别，右轮是否保持不动，指示灯是否在结束后全熄灭，以及两块 MCU 是否复位。
5. 验收结论只标记本次左轮空载验证，不扩大为双轮或底盘验收。
6. 检查记录不含凭据，只暂存该文件，自动提交并推送 `main`。

## 安全边界

- 本计划总共最多发送一笔非零 `MOVE_WHEELS`，右轮步数恒为零。
- 此次已确认设计不代替 Task 3 的当次现场确认。
- AI 和普通网页运动接口不参与探针，也不获得解除 ESTOP 的自动权限。
- 未得到物理观察结果前，不测试反向、右轮或双轮。
