# 左轮首次带电验证设计

## 1. 背景

机器人已完成 ESP32-S3 与 STM32F103 之间的 SPI 通信、状态控制和 ESTOP 恢复验证，但双 28BYJ-48 电机尚未进行真机转动校准。左侧 ULN2003 的 `IN1-IN4` 依次连接 STM32 `PA8、PA9、PA10、PA11`；右轮使用独立的 GPIO，本次不动作。

现场条件已确认：

- 左右轮离地悬空。
- 可以立即断开两块 ULN2003 的 5 V 电机供电。
- ESP32、STM32、两块 ULN2003 与升压电源负极已共地。
- 电机电流回路不经过单片机细地线，5 V 不进入 GPIO。

## 2. 目标

1. 用一次受限动作验证左轮电机、ULN2003、接线和现有半步序列能否正常工作。
2. 记录正步数对应的实际转动方向，作为后续左右轮方向标定的依据。
3. 验证动作正常完成、超时或中止后，固件都会释放左右电机线圈。

## 3. 非目标

- 本次不测试反转、右轮、双轮联动、负载运动或长时间循环。
- 不新增网页校准 API、按钮或 AI 工具，不扩大远程运动权限。
- 不在首次转动时修改相序、方向翻转或运动上限。
- 不将一次短时空载成功视为底盘已完成验收。

## 4. 执行架构

使用只在 USB 调试期间运行的一次性 MicroPython 探针，复用正式运行时已验证的调用链：

```text
操作员当次确认
  -> USB / MicroPython 受限探针
  -> DeviceService
  -> ESP32 SPI 链路
  -> STM32 协议与状态机
  -> motion_service
  -> 左侧 ULN2003 / 28BYJ-48
```

探针不部署到正式网页路由，也不向 `robot_tools` 注册。右轮步数始终为零；STM32 每次应用左轮相位时，仍明确向右侧 ULN2003 输出全零。

## 5. 固定参数

| 参数 | 值 |
| --- | ---: |
| 左轮步数 | `+64` half-steps |
| 右轮步数 | `0` |
| 最大速度 | `120 half-steps/s` |
| 加速度 | `300 half-steps/s²` |
| STM32 动作超时 | `1500 ms` |
| 自动重试 | `0` |
| 执行次数 | `1` |

这些参数低于当前固件的 `800 half-steps/s`、`1200 half-steps/s²` 和 `2000 ms` 硬上限。本次探针不接受外部速度或步数参数，避免现场误输入放大动作。

## 6. 执行流程

1. 操作员再次确认轮子悬空、ULN2003 5 V 断电手段可用，然后明确说“已准备”。
2. 探针启动 `DeviceService` 的连续 SPI 轮询，等待 STM32 回应并取得状态快照。
3. 如果切换至 USB 调试导致心跳中断并进入 ESTOP，只在链路已恢复、没有活动运动且本次操作员确认仍有效时，执行一次 `CLEAR_ESTOP`。不解除 FAULT。
4. 要求状态为 IDLE、故障码为零、链路健康，然后请求 `MANUAL` 模式。
5. 发送一次固定的 `MOVE_WHEELS(+64, 0, 120, 300, 1500)`，等待 ACK 和最终运动结果。
6. 只有收到 `DONE` 才记为协议成功；正常完成后请求 IDLE。运动服务在 `DONE`、超时和中止时均输出左右全零，释放线圈。
7. 读取最终状态，停止探针。恢复 ESP32 正式应用后，再恢复 STM32 通信会话；不在这个恢复步骤中发送运动命令。如果切换期间再次触发 ESTOP，保留锁定，等正式网页恢复后由操作员重新检查并二次确认解除，收尾流程不自动解除。

## 7. 异常处置

下列任一情况都立即中止流程：

- STM32 未连接，或初始状态不能安全回到 IDLE。
- `SET_MODE`、`MOVE_WHEELS` 被拒绝或超时。
- 未收到 `DONE`，收到超时、中止或未知结果。
- ESP32 或 STM32 重启，显示异常，或电机出现持续抖动、异响、不转。

如果通信仍可用，探针发送一次 `STOP`，由 STM32 锁定 ESTOP 并释放线圈；探针不自动解除此 ESTOP，也不重试动作。操作员同时断开 ULN2003 5 V。若已断联，依赖 STM32 的 750 ms 心跳超时释放线圈，但仍立即执行物理断电，不等待软件结果。

如果只抖动或不转，优先检查 `IN1-IN4` 顺序、电机插头、5 V 压降和共地；本次不通过随机调换软件相序来掩盖接线问题。

## 8. 验收与记录

本次验证通过需同时满足：

1. 只有左轮以连续半步方式平稳转过小而可见的角度，右轮完全不动。
2. 动作在 1500 ms 内结束，协议返回 `DONE`。
3. 结束后两块 ULN2003 的相位指示灯全部熄灭，没有持续吸合、嗡鸣或发热。
4. ESP32 和 STM32 未重启，最终状态与动作结果一致。

执行后必须暂停，由操作员报告 `向前`、`向后`、`抖动` 或 `不动` 之一，并把协议结果、初末状态、物理观察及是否发生复位写入 `docs/testing/m1-left-wheel-bringup.md`。未完成此复核前，不执行反转或右轮动作。
