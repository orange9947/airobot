# 右轮最后一种独立相序诊断设计

## 依据

驱动板正反面照片确认：模块使用 ULN2003A，P3 白色插座将 A-D 四路输出和 VCC 接到 28BYJ-48 五线插头；电机为 5V 型，红色公共线位置正确。本地照片仅用于诊断，不加入 Git。

四个线圈忽略旋转方向后只有三种不同的环形排列。标准 `1-2-3-4` 与中间相交换后的 `1-3-2-4` 都只产生轻微抖动。最后尚未验证的独立排列是 `1-2-4-3`；验证它可以穷尽相序排列，而不是随机试线。

## 临时接线

电机 5V 断开时，只在右侧 ULN2003 上交换 IN3 与 IN4：

| STM32 | 临时连接 |
| --- | --- |
| PB0 | IN1 |
| PB1 | IN2 |
| PB10 | IN4 |
| PB11 | IN3 |

左侧接线、两侧 VCC/GND 和电机五线插头不变。接线牢固且无短路后才接回 5V。

## 固定动作

一次性 USB 探针只发送一次 `MOVE_WHEELS(0, +64, 120, 300, 1500)`。探针沿用连接、ESTOP、故障、活动命令与动作结果门禁，不接受参数、不自动重试。动作时左板必须全程不亮，结束或异常均释放两侧输出。

## 结果

- 若右侧裸轴连续转动，立即断开 5V。随后恢复 IN1-IN4 直连接线，在 STM32 板级 ULN2003 映射层永久采用逻辑顺序 `1-2-4-3`，不污染运动服务和协议。
- 若右侧仍抖动或不动，立即断开 5V 并恢复 `PB10→IN3、PB11→IN4`。三种独立相序全部排除，下一步只能逐相测量 P3 上 A-D 输出到红色公共线的实际线圈压差。
- 本次不测试左轮、反转、双轮动作或更高参数。
