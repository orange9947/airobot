# 左电机带载电压诊断设计

## 目标

左轮 `+64` 半步测试已确认：左侧 ULN2003 相位灯依次亮、右侧全程断电、动作后两侧全部释放，但左轮没有可见运动。上一次准备测量带载电压时表笔松脱，因此需要一次更慢、位移更小的受限动作，为万用表提供稳定读数窗口。

本次只测左侧 ULN2003 `VCC-GND` 的最低带载电压，不以轮子位移或声音作为验收目标，不修改正式固件、网页或运动上限。

## 固定参数

| 参数 | 值 |
| --- | ---: |
| 左轮步数 | `+10` half-steps |
| 右轮步数 | `0` |
| 最大速度 | `10 half-steps/s` |
| 加速度 | `10 half-steps/s²` |
| STM32 超时 | `2000 ms` |
| 自动重试 | `0` |
| 执行次数 | `1` |

按当前运动算法，第一步约在 399 ms 出现，线圈带载窗口约 1500 ms，动作约在 1899 ms 正常完成。输出轴理论位移约 0.88 度。

## 执行与安全

使用一次性 USB MicroPython 探针复用正式 `DeviceService` 调用链。探针只在 STM32 已连接、无活动命令、无不可恢复故障时继续；允许清除 USB 切换造成的链路丢失 ESTOP，然后进入 MANUAL 并发送唯一一笔固定动作。

执行前必须确认万用表表笔固定在左侧 ULN2003 `VCC-GND`、轮子悬空、两板指示灯全灭且可以立即断开 5V。动作时左板低速换相，右板必须全程不亮。出现明显电压崩落、右板亮灯、持续异响、指示灯不灭或 MCU 复位时立即断开 5V。

探针只接受协议 `DONE`、左 10 步和右 0 步为成功。任何拒绝、超时或异常都尝试 STOP 并保留 ESTOP，不自动重试。结束后恢复 ESP32 正式程序并删除临时探针。

## 结果判断

- 记录动作前电压和动作期间最低电压。
- 若带载电压明显低于空载值，优先检查电池输出、升压模块能力、接头和供电线压降。
- 若带载电压稳定而电机仍无动作，下一步检查 ULN2003 输出端到电机公共线之间的实际线圈压差和插头接触。
- 无论结果如何，本次不增加步数、速度或持续时间。
